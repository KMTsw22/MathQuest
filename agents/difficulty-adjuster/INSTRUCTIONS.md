# Difficulty Adjuster — Instructions

## Step 1: 현재 분포 분석

입력된 `problems[]`에서 `difficulty` 필드 값을 집계한다.

```
현재: { easy: 4, medium: 4, hard: 2 }
목표: { easy: 3, medium: 5, hard: 2 }
→ Easy 1개를 Medium으로 변형 필요
```

---

## Step 2: 난이도 변형 기법

### Easy → Medium 변형
1. **숫자 범위 확대**: 단순 분수(1/2, 1/4) → 일반 분수(3/8, 5/6)
2. **연산 단계 추가**: 1단계 계산 → 2단계 계산 요구
3. **맥락 복잡화**: 직접 질문 → 조건이 하나 더 있는 질문

### Medium → Hard 변형
1. **역산 요구**: 결과 주고 원래 값 찾기
2. **혼합 연산**: 두 가지 연산 조합
3. **세 자리 수 / 혼합 분수 도입**
4. **비교 추론**: 단순 계산이 아닌 관계 판단 요구

### Medium → Easy 변형
1. **숫자 단순화**: 복잡한 분수 → 단위분수(1/n)
2. **연산 단계 제거**: 2단계 → 1단계
3. **힌트 제공**: 풀이 방향 암시하는 문장 추가

### Hard → Medium 변형
1. **숫자 범위 축소**
2. **역산 → 순산으로 전환**
3. **혼합 연산 분리**

---

## Step 3: 변형 제약 조건

- 변형 후 문제가 해당 **학년 CCSS 범위 내**에 있어야 함
- 변형 후 **정답과 풀이 단계를 반드시 업데이트**
- 원래 문제 유형(word_problem, equation 등)을 유지하는 것을 원칙으로 함
  (단, visual → equation 변환은 허용)
- 변형 내용을 `adjustment_note`에 한 줄로 기록

---

## Step 4: 검증 후 반환

변형 완료 후 최종 분포 확인:
```
최종: { easy: 3, medium: 5, hard: 2 } ← 목표와 일치
```

일치하면 Validator로 전달.
불일치하면 Step 2 재수행 (최대 2회).
